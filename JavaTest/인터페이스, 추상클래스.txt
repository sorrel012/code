//인터페이스, Interface
	- 멤버 변수 선언 불가능
	- 추상 메소드를 멤버로 가진다. > ★★★★추상 메소드들의 집합
	- 접근 지정자는 반드시 public으로 선언한다.
	- 인터페이스는 구현 멤버를 가질 수 없다.
		※구현 멤버 : 변수, 구현부를 가지는 메소드
	- 클래스가 인터페이스를 상속 받아 구현해야 한다.
		> 자식 클래스의 의무!
	- 자바는 기본적으로 다중 상속을 지원하지 않지만, 인터페이스는 다중 상속이 가능하다.
	- 인터페이스를 상속한다는 표현보다는 구현한다는 표현을 주로 사용한다.
	

//추상 클래스, Abstract class
	- 구현 멤버를 가질 수 있다.(= 일반 클래스의 성질)
	
	
//상속 관계

	(부모)		(자식)
	클래스		클래스	
	인터페이스		클래스	> 상속된 추상 클래스를 자식이 구현해야 한다.
	추상클래스		클래스	> 구현 멤버는 그대로 상속 받고, 추상 멤버는 구현한다.
	인터페이스		인터페이스
	추상클래스		추상클래스	> 상속 받은 추상 메소드를 구현해도 되고 안 해도 된다.
	인터페이스		추상클래스

	- 일반 클래스는 일반 클래스의 부모 역할만 가능하다.
	- 일반 클래스는 인터페이스나 추상클래스의 부모 역할이 불가능하다.
	
	- 추상클래스는 추상클래스, 일반 클래스의 부모 역할을 한다.
	- 추상클래스는 인터페이스 부모 역할이 불가능하다.
	
	- 인터페이스는 인터페이스, 추상클래스,일반 클래스의 부모 역할을 한다.
	
		(상위)										(하위)
	- 인터페이스 -> 인터페이스 -> 추상클래스 -> 추상클래스 -> 일반 클래스
